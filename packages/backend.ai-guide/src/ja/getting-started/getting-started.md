# 始めに

このセクションでは、Backend.AIの使用を開始する方法について簡潔な概要を提供します。新しいインストールを設定する場合でも、既存のデプロイメントを探索する場合でも、これらのステップは立ち上げと実行に役立ちます。

## 概要

Backend.AIは、いくつかの主要コンポーネントで構成されています：

- **Manager**: リソースとセッションを管理する中央制御プレーン
- **Agent**: コンテナ（セッション）を実行するコンピュートノード
- **WebUI**: インタラクティブアクセス用のWebベースユーザーインターフェース
- **Storage Proxy**: ユーザーデータの永続ストレージを処理

## クイックスタートチェックリスト

Backend.AIを開始するには：

1. **デプロイメント方法の選択**
   - Docker Compose（開発/テスト環境に推奨）
   - Kubernetes（本番環境に推奨）
   - ソースインストール（上級ユーザー向け）

2. **システム要件**
   - Linuxベースシステム（Ubuntu 20.04+推奨）
   - DockerとDocker Compose
   - 最低4GB RAMと2つのCPUコア
   - コンテナイメージとユーザーデータ用のストレージ

3. **初期セットアップ**
   - Backend.AIコンポーネントのインストール
   - ストレージとネットワーキングの設定
   - 初期ユーザーアカウントの設定
   - インストールの検証

4. **最初のセッション**
   - WebUIへのアクセス
   - コンピュートセッションの作成
   - 簡単なPythonスクリプトの実行
   - 利用可能な環境の探索

## 次のステップ

Backend.AIが動作したら：

- WebUIを探索してインターフェースに慣れ親しんでください
- さまざまなプログラミング言語でセッション作成を試してみてください
- 永続データ用のストレージフォルダについて学習してください
- 必要に応じて追加のユーザーとプロジェクトを設定してください

詳細なインストール手順については、このガイドのインストールセクションに進んでください。
