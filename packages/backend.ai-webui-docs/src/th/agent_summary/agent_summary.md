# สรุปเอเจนต์

หน้าสรุปเอเจนต์ให้ภาพรวมแบบอ่านอย่างเดียวของ**เอเจนต์**การคำนวณ (โหนดเอเจนต์) ที่โปรเจกต์ของคุณสามารถใช้งานได้ และสถานะการจัดสรรทรัพยากรของแต่ละเอเจนต์ในปัจจุบัน ใช้หน้านี้เพื่อตรวจสอบว่ามีความจุของ CPU, หน่วยความจำ และตัวเร่งความเร็ว (accelerator) เหลืออยู่เท่าใดก่อนที่จะสร้างเซสชันการคำนวณ

หน้านี้จะปรากฏภายใต้กลุ่มเมนู **Metrics** เมื่อการกำหนดค่า `hideAgents` ของเซิร์ฟเวอร์ถูกปิดใช้งาน ตั้งแต่เวอร์ชัน 22.09 เป็นต้นไป Backend.AI WebUI ยังสามารถแสดงข้อมูลบางส่วนของโหนดเอเจนต์ให้กับผู้ใช้ทั่วไปได้เมื่อมีการกำหนดค่าเซิร์ฟเวอร์ให้อนุญาต

## การกรองและสถานะ

แถบเครื่องมือด้านบนของรายการช่วยให้คุณจำกัดเอเจนต์ที่จะแสดงได้

- **การสลับสถานะ**: สลับระหว่างเอเจนต์ที่ **เชื่อมต่อแล้ว** และ **ยุติแล้ว** รายการจะแสดงเอเจนต์ที่ **เชื่อมต่อแล้ว** เป็นค่าเริ่มต้น
- **ID**: กรองรายการตาม ID ของเอเจนต์
- **จัดตารางได้**: กรองตามว่าเอเจนต์สามารถรับเซสชันใหม่ได้หรือไม่ โดยเลือก **เปิดใช้งาน** หรือ **ปิดใช้งาน**
- **รีเฟรช**: คลิกปุ่ม **รีเฟรช** เพื่อโหลดรายการใหม่ด้วยข้อมูลเอเจนต์ล่าสุด

## คอลัมน์

รายการเอเจนต์จะแสดงคอลัมน์ต่อไปนี้

- **ID**: ตัวระบุเฉพาะของเอเจนต์ คอลัมน์นี้สามารถจัดเรียงได้
- **สถาปัตยกรรม**: สถาปัตยกรรม CPU ของเอเจนต์ (เช่น `x86_64`)
- **การจัดสรร**: ความจุที่ใช้ไปและที่ยังว่างอยู่ของแต่ละสล็อตทรัพยากรบนเอเจนต์ แสดงพร้อมกับแถบความคืบหน้าของการใช้งาน
   * CPU จะแสดงเป็นจำนวนคอร์ หน่วยความจำเป็นหน่วย GiB และตัวเร่งความเร็วใด ๆ (เช่น GPU หรือ fGPU) ที่มีอยู่บนเอเจนต์จะแสดงเป็นสล็อตเพิ่มเติม
   * แถบความคืบหน้าจะเปลี่ยนเป็นสีแดงเมื่อการใช้งานเกิน 80%
- **กลุ่มทรัพยากร**: กลุ่มทรัพยากรที่เอเจนต์สังกัดอยู่ คอลัมน์นี้สามารถจัดเรียงได้
- **จัดตารางได้**: ว่าเอเจนต์สามารถรับเซสชันใหม่ได้หรือไม่ ไอคอนเครื่องหมายถูกแสดงว่าเอเจนต์จัดตารางได้ ส่วนไอคอนเครื่องหมายลบที่จางลงแสดงว่าไม่สามารถจัดตารางได้ คอลัมน์นี้สามารถจัดเรียงได้

คุณสามารถปรับแต่งว่าจะแสดงคอลัมน์ใดและลำดับของคอลัมน์ได้โดยใช้ไอคอนรูปเฟืองตั้งค่าคอลัมน์ที่มุมขวาล่างของตาราง คอลัมน์ที่ซ่อนไว้และลำดับคอลัมน์จะถูกจดจำไว้สำหรับการเข้าชมครั้งถัดไป

:::note
เอเจนต์ที่สังกัดกลุ่มทรัพยากรเฉพาะสำหรับ SFTP จะถูกแยกออกจากรายการนี้โดยอัตโนมัติ ดังนั้นเอเจนต์ที่ใช้สำหรับจัดเก็บข้อมูลเท่านั้นจะไม่ปรากฏที่นี่
:::

หากต้องการวางแผนการจัดสรรทรัพยากรก่อนเริ่มเซสชัน ให้ตรวจสอบความจุที่ว่างอยู่ที่นี่ จากนั้นไปยัง[หน้าเซสชัน](#session-page)
