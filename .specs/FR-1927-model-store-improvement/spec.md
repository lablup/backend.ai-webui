# Model Store 개선

> **Epic**: FR-1927
> **Spec Task**: FR-2382
> **Status**: Draft
> **Date**: 2026-03-26

## 개요

Model Store 경험을 세 가지 축으로 개선합니다: (1) 스토리지 접근성 기반으로 사용자에게 실제 사용 가능한 모델만 보여주는 플로우, (2) 모델 아이템 목록의 UI를 깔끔한 리스트 형태로 리디자인, (3) 모델 폴더에서 노티피케이션 기반 서비스 실행 플로우 + Model Store에서 "Clone & Start Service" 통합 플로우.

## 문제 정의

**사용자 측 문제:** Model Store가 현재 모든 모델 아이템을 보여주지만, 사용자의 프로젝트에서 해당 스토리지를 마운트할 수 없는 경우 클론이 실패합니다. 스토리지 접근성에 따른 필터링이 없습니다.

**UI 문제:** 현재 모델 아이템 목록이 썸네일 카드 방식으로 표시되어 정보 밀도가 낮고 한눈에 파악하기 어렵습니다. 깔끔한 리스트 형태로 개선이 필요합니다.

**서비스 실행 플로우 문제:** 클론과 서비스 실행이 별도 액션으로 나뉘어 있어 사용자가 여러 단계를 수동으로 거쳐야 합니다. 클론된 폴더를 이름 기반으로 매칭하는 방식이라 중복 문제가 있고, 클론 후 service definition 존재 여부를 확인하려면 개별 API 호출이 필요해 목록에서 표현할 수 없습니다.

**성능 문제:** 현재 모델 목록 조회 시 GQL 응답이 약 2MB로 매우 크고, 페이지네이션이 없어 로딩이 느립니다.

## 사용자 스토리

### 공통: 모델 아이템 목록 UI 개선

- 사용자/관리자로서, 모델 아이템 목록을 썸네일 카드가 아닌 깔끔한 리스트 형태로 보고 싶다. HuggingFace 모델 목록과 유사하게 아이콘 + 모델명 + 태그 형태로 정보를 빠르게 파악할 수 있도록.
- 사용자/관리자로서, 주요 모델 제조사/브랜드 아이콘(icons.lobehub.com 활용)이 목록에 표시되면 좋겠다. 모델을 시각적으로 빠르게 식별할 수 있도록.
- 일반 사용자로서, 모델 폴더의 내용(README 등)을 조회할 수 있어야 한다. 모델 정보를 클론 전에 확인할 수 있도록.

### 사용자: 모델 서비스 실행

- 사용자로서, 현재 프로젝트에서 마운트 가능한 스토리지의 모델 아이템만 보고 싶다. 항상 성공적으로 클론하고 사용할 수 있도록.
- 사용자로서, 모델 폴더에서 서비스 실행 버튼을 누르면 노티피케이션을 통해 단계별 진행 상태를 확인하고, 결과에 따른 다음 액션을 안내받고 싶다.
- 사용자로서, Model Store에서 "Clone & Start Service" 버튼 하나로 클론부터 서비스 실행까지 한 번에 진행하고 싶다. 이미 클론한 폴더가 있다면 폴더 목록에서 클론 없이 실행할 수 있다는 안내를 받고 싶다.

## 요구사항

### Must Have

- [ ] 일반 사용자에게 스토리지 접근성 기반으로 모델 목록이 필터링되어야 한다
  - 현재 프로젝트에서 마운트 가능한 스토리지의 모델 아이템만 표시
  - 사용자 개입 없이 자동 필터링
- [ ] 모델 아이템 목록을 썸네일 카드 방식에서 깔끔한 리스트 형태로 변경해야 한다
  - 기존 썸네일/카드 UI를 제거하고 HuggingFace 모델 목록과 유사한 리스트 레이아웃으로 교체
  - 모델명 + 태그(framework, task 등 model definition 메타데이터 기반) 표시
  - 한줄평(description)은 관리 불가하므로 표시하지 않음
  - 모델 제조사/브랜드 아이콘은 icons.lobehub.com을 활용하여 모델 이름 키워드 기반으로 매핑 (예: GPT/OpenAI → OpenAI 아이콘, Llama → Meta 아이콘, Gemma → Google 아이콘). 매핑 없는 모델은 기본 아이콘 표시
- [ ] 모델 폴더에서 노티피케이션 기반 서비스 실행 플로우를 구현해야 한다
  - 모델 폴더 옆에 서비스 실행 버튼 추가 (모델 서빙 메뉴 아이콘과 동일)
  - 버튼 클릭 시 노티피케이션으로 단계별 진행 상태 안내 (제목: "모델 서비스 준비 중")
  - 단계: model definition 확인 → service definition 확인 → 결과에 따라 분기:
    - **둘 다 있음** → 서비스 생성 요청 자동 호출 → "서비스 생성이 요청되었습니다" + 서비스 디테일로 가기 링크
    - **service definition 없음** → "서비스 데피니션 파일이 없으므로 서비스 런처에서 시작 가능합니다" → 서비스 런처로 자동 이동 (워닝 표시: "서비스 데피니션이 없어서 직접 설정해야 합니다")
    - **model definition도 없음** → "모델 데피니션이 필요합니다" + 폴더 열기 링크
- [ ] Model Store에서 "Clone & Start Service" 통합 버튼으로 서비스 실행 플로우를 개선해야 한다
  - 클론과 실행을 하나의 버튼으로 통합
  - 버튼을 누르면 모달 표시, 모달 상단에 안내: "이미 클론한 폴더가 있다면 **폴더 목록**에서 클론 없이 서비스 실행할 수 있습니다." ("폴더 목록"은 모델 타입 필터가 선택된 데이터 목록 페이지로의 링크)
  - 복제 시 폴더 이름을 고정 포맷으로 생성 (랜덤 스트링 사용 안 함), 사용자에게 폴더 이름 확인받음
  - 복제 시작 후 노티피케이션으로 진행 상태 안내 (위 폴더 디테일 플로우와 동일한 단계별 로직, 앞에 클론 단계만 추가)
    - 클론 중 → model definition 확인 → service definition 확인 → 결과에 따라 분기 (위와 동일)
- [ ] 일반 사용자도 모델 폴더의 내용을 조회할 수 있어야 한다
  - 현재 관리자에게만 보이는 모델 폴더 링크를 일반 사용자에게도 표시
  - 모델 디테일에서 README 등 내용 조회 가능
- [ ] 모델 아이템 목록에 페이지네이션을 적용해야 한다
  - 현재 전체 목록을 한 번에 불러오는 방식에서 페이지네이션으로 전환
  - GQL 응답 크기 문제(약 2MB) 해결
- [ ] 스토리지 기반 필터링을 지원하는 Model Store 아이템 조회 Backend API `TODO(needs-backend)`
- [ ] Model Store 아이템 목록 페이지네이션 지원 Backend API `TODO(needs-backend)`

## 인수 조건

### 모델 아이템 목록 UI

- [ ] 모델 아이템이 HuggingFace 모델 목록과 유사한 리스트 형태로 표시된다 (아이콘 + 모델명 + 태그)
- [ ] 모델 제조사/브랜드별 아이콘이 icons.lobehub.com 기반으로 표시된다 (매핑 없는 모델은 기본 아이콘)
- [ ] 태그(framework, task 등)가 model definition 메타데이터에서 추출되어 표시된다
- [ ] 목록에 페이지네이션이 적용되어 있다

### 스토리지 기반 필터링 (사용자)

- [ ] 일반 사용자가 Model Store를 볼 때 현재 프로젝트에서 마운트 가능한 스토리지의 모델만 보인다
- [ ] 프로젝트 전환 시 새 프로젝트의 접근 가능한 스토리지에 맞게 모델 목록이 업데이트된다
- [ ] 사용자에게 표시된 모든 모델은 성공적으로 클론할 수 있다 (스토리지 접근 불가로 인한 오류 없음)

### 서비스 실행 (사용자 — 폴더 디테일)

- [ ] 모델 타입 폴더 옆에 서비스 실행 버튼이 있다 (모델 서빙 메뉴 아이콘과 동일)
- [ ] 버튼 클릭 시 노티피케이션이 나타나고 "모델 서비스 준비 중" 제목으로 단계별 진행 상태가 표시된다
- [ ] model definition + service definition 둘 다 있으면 → 서비스 생성 요청이 자동 호출되고 "서비스 생성이 요청되었습니다" + 디테일로 가기 링크가 표시된다
- [ ] service definition이 없으면 → 서비스 런처로 자동 이동하며, 워닝으로 "서비스 데피니션이 없어서 직접 설정해야 합니다"가 표시된다
- [ ] model definition도 없으면 → "모델 데피니션이 필요합니다" + 폴더 열기 링크가 표시된다

### 서비스 실행 플로우 (사용자 — Model Store)

- [ ] Model Store에 "Clone & Start Service" 버튼이 있다
- [ ] 버튼을 누르면 모달이 나타나고, 상단에 "이미 클론한 폴더가 있다면 **폴더 목록**에서 클론 없이 서비스 실행할 수 있습니다." 안내가 표시된다 ("폴더 목록"은 모델 타입 필터가 선택된 데이터 목록 페이지로의 링크)
- [ ] 폴더 이름을 확인하는 단계가 있다 (고정 포맷, 랜덤 스트링 없음)
- [ ] 확인 후 클론이 시작되며 노티피케이션으로 단계별 진행 상태가 안내된다 (클론 중 → model definition 확인 → service definition 확인 → 결과 분기)
- [ ] 결과 분기는 폴더 디테일의 서비스 실행 플로우와 동일하다
- [ ] 중간 과정에서 오류 발생 시 명확한 에러 메시지가 노티피케이션에 표시된다

### 모델 폴더 조회 (일반 사용자)

- [ ] 일반 사용자에게도 모델 폴더 링크가 표시된다 (현재 관리자에게만 표시됨)
- [ ] 일반 사용자가 모델 폴더 내용(README 등)을 조회할 수 있다

## 범위 외 (Out of Scope)

- HuggingFace 모델 임포트 플로우 (별도 스펙으로 분리)
- 관리자 전용 Model Store 아이템 관리 페이지 (별도 스펙으로 분리)
- 이미 임포트된 모델의 버전 관리 또는 제자리 업데이트
- 크로스 스토리지 모델 클론
- service definition 자체의 포맷이나 스키마 변경
- 폐쇄망(air-gapped) 환경에서의 HuggingFace 연동
- 모델 디테일에서 채팅 기능 (서비스가 띄워져 있을 때 채팅 UI)
